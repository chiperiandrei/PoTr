1. Cum utilizam API-ul?

    REST API-ul disponibil are dezvoltate urmatoarele ENDPOINT-uri

  •          GET request
      /api/users - listeaza toti utilizatorii aplicatiei ( doar datele ce nu pot fi utilizate pentru logare)
      /api/poems - listeaza toate poemele din baza de date a aplicatiei
      /api/authors - listeaza toti autorii din baza de date a aplicatiei
  •          POST request
      /api/poems - insereaza poeme in cadrul aplicatiei prin intermediul JSON-ului dat ca body.
      Exemplu de JSON  [ { "TITLE": "Example of title", "AUTHOR_ID": "1", "LANGUAGE": "RO", "STAFF_ID": "1" } ]
      /api/authors - insereaza autori in cadrul aplicatiei prin intermediul JSON-ului dat ca body.
      Exemplu de JSON  [ { "NAME": "Name of the author", "BIRTH_DATE": "1234", "DEATH_DATE": "4321" } ]
  •          PUT request
      /api/users/{id} - actualizeaza nume si prenumele unui utilizator prin inntermediul id-ului dat ca parametru si al JSON-ului dat ca body
      Exemplu de JSON  [ { "FIRST_NAME": "Nume1", "LAST_NAME": "Nume2" } ]
      Pot lipsi campuri, dar nu pot fi mai multe decat cele doua mentionate. ( FIRST_NAME, LAST_NAME)
      /api/poems/{id} - actualizeaza datele unui poem prin inntermediul id-ului dat ca parametru si al JSON-ului dat ca body
      Exemplu de JSON  [ { "TITLE": "Liceu", "LANGUAGE": "RO", "ID_AUTHOR": "1", "ID_STAFF": "2" } ]
      Pot lipsi campuri, dar nu pot fi mai multe decat cele doua mentionate. ( TITLE, LANGUAGE,ID_AUTHOR,ID_STAFF)
      /api/authors/{id} - actualizeaza informatiile despre un autor prin inntermediul id-ului dat ca parametru si al JSON-ului dat ca body
      Exemplu de JSON  [ { "NAME": "Vasile Alessaacsandri", "BIRTH_DATE": "1821", "DEATH_DATE": "1890" } ]
      Pot lipsi campuri, dar nu pot fi mai multe decat cele doua mentionate. ( NAME, BIRTH_DATE,DEATH_DATE)
  •          DELETE request
      /api/users/{id} - sterge utilizatorul dat prin parametrul {id}
      /api/poems/{id} - sterge poemul dat prin parametrul {id}
      /api/authors/{id} - sterge autorul dat prin parametrul {id}
